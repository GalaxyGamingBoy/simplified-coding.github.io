# Μάθημα 1ο: Εισαγωγή στο Git & GitHub

## Τι είναι το Git και γιατί είναι σημαντικό;

Το Git είναι ένα σύστημα διαχείρισης εκδόσεων (Version Control System - VCS)
που χρησιμοποιείται ευρέως από προγραμματιστές για την παρακολούθηση και τον έλεγχο της ανάπτυξης λογισμικού.
Είναι σχεδιασμένο να αποθηκεύει, να διαχειρίζεται και να καταγράφει τις αλλαγές στον κώδικα πηγαίου κώδικα κατά τη διάρκεια της ανάπτυξης λογισμικού.

Κάποια από τα βασικά χαρακτηριστικά του Git περιλαμβάνουν:

1. **Ιστορικό Αλλαγών:** Το Git διατηρεί αναλυτικό ιστορικό όλων των αλλαγών που έχουν γίνει στον κώδικα.

2. **Διακλαδώσεις (Branches):** Οι προγραμματιστές μπορούν να δημιουργήσουν διακλαδώσεις από τον κώδικα πηγαίου κώδικα,
επιτρέποντάς τους να εργαστούν παράλληλα σε διάφορες λειτουργίες χωρίς να επηρεάζουν τον κώδικα του κύριου έργου.

3. **Συγχώνευση (Merge):** Το Git επιτρέπει τη συγχώνευση των διακλαδώσεων, δηλαδή την ενσωμάτωση των αλλαγών από μια διακλάδωση πίσω στον κύριο κώδικα.

4. **Απομακρυσμένα Αποθετήρια (Remote Repositories):** Το Git υποστηρίζει τη συνεργασία μεταξύ ανθρώπων που βρίσκονται σε διάφορες τοποθεσίες.
Ο κώδικας αποθηκεύεται σε απομακρυσμένα αποθετήρια, όπου μπορεί να χρησιμοποιηθεί από πολλούς προγραμματιστές.

Η σημασία του Git αναδεικνύεται από τα εξής:

1. **Εύκολη Συνεργασία:** Επιτρέπει σε προγραμματιστές από διάφορες ομάδες και τοποθεσίες να συνεργαστούν αποτελεσματικά στον κώδικα χωρίς σύγχυση.

2. **Παρακολούθηση Αλλαγών:** Το Git επιτρέπει την ακριβή παρακολούθηση όλων των αλλαγών που έχουν γίνει στον κώδικα, κάτι που βοηθάει στην εύρεση σφαλμάτων
και τη διαχείριση του έργου.

3. **Ασφάλεια και Ακεραιότητα:** Ο κώδικας αποθηκεύεται ασφαλώς και μπορεί να ανακτηθεί σε περίπτωση απώλειας δεδομένων ή σφάλματος.

4. **Ελευθερία για Δοκιμές:** Οι προγραμματιστές μπορούν να δημιουργούν προσωρινές αλλαγές και να τις δοκιμάζουν χωρίς να επηρεάζουν τον κύριο κώδικα.

Συνολικά, το Git αποτελεί ένα απαραίτητο εργαλείο για τον ανάπτυξη λογισμικού και τη διαχείριση του κώδικα,
επιτρέποντας αποτελεσματική συνεργασία και παρακολούθηση των αλλαγών.

## Τι είναι το GitHub και πως συγκρίνεται με το Git;

Το GitHub είναι μια πλατφόρμα διαχείρισης κώδικα που χρησιμοποιείται για τη συνεργασία σε έργα λογισμικού που χρησιμοποιούν το Git.
Το GitHub έχει πολλές λειτουργίες και εργαλεία που βοηθούν στη διαχείριση του κώδικα και την επικοινωνία μεταξύ των προγραμματιστών μαζί με το Git.
Μερικά απο αυτά είναι:

1. **Διαχείριση Κώδικα:** Το Git είναι το σύστημα διαχείρισης εκδόσεων που χρησιμοποιείται για την παρακολούθηση και τον έλεγχο του κώδικα πηγαίου κώδικα.
Το GitHub παρέχει μια πλατφόρμα όπου οι προγραμματιστές και η ομάδα ανάπτυξης μπορούν να αποθηκεύσουν, να διαχειριστούν και να συνεργαστούν σε έργα Git.

2. **Αποθετήρια (Repositories):** Στο GitHub, ο κώδικας αποθηκεύεται σε αποθετήρια, τα οποία είναι χώροι όπου οι προγραμματιστές αποθηκεύουν τον κώδικά τους.
Τα αποθετήρια GitHub χρησιμοποιούν το Git για να παρακολουθούν και να ελέγχουν τις αλλαγές στον κώδικα.

3. **Συνεργασία:** Ένα από τα βασικά πλεονεκτήματα του GitHub είναι η δυνατότητα συνεργασίας.
Προγραμματιστές από όλον τον κόσμο μπορούν να συνεισφέρουν στον κώδικα πηγαίου κώδικα, να προτείνουν αλλαγές και να σχολιάζουν τον κώδικα.

4. **Παρακολούθηση Έργων:** Το GitHub παρέχει εργαλεία για την παρακολούθηση της εξέλιξης των έργων,
τη διαχείριση των εκδόσεων και τη δημιουργία ζητήσεων ενσωμάτωσης (Pull Requests) για την ενσωμάτωση νέων αλλαγών.

Συνολικά, το GitHub διευκολύνει τη συνεργασία, την κοινοποίηση και τη διαχείριση του κώδικα πηγαίου κώδικα σε ομάδες ανάπτυξης και κοινότητες προγραμματιστών.

## Αρχικά βήματα με το Git και το GitHub

### Εγκατάσταση Git

#### Windows

Για την εγκατάσταση του Git σε υπολογιστές με Windows,
μπείτε στην [επίσημη ιστοσελίδα του Git](https://git-scm.com/download/win) και κατεβάστε το αρχείο εγκατάστασης.

#### Mac OS

Για την εγκατάσταση του Git σε υπολογιστές με Mac OS,
μπείτε στην [επίσημη ιστοσελίδα του Git](https://git-scm.com/download/mac) και κατεβάστε το αρχείο εγκατάστασης.

#### Linux

Για την εγκατάσταση του Git σε υπολογιστές με Linux,
τρέξτε τη σωστή εντολή για τη διανομή σας:

##### Debian/Ubuntu
`apt-get install git`

##### Fedora
`yum install git (up to Fedora 21)`
`dnf install git (Fedora 22 and later)`

##### Gentoo
`emerge --ask --verbose dev-vcs/git`

##### Arch Linux
`pacman -S git`

##### openSUSE
`zypper install git`

##### Mageia
`urpmi git`

##### Nix/NixOS
`nix-env -i git`

##### FreeBSD
`pkg install git`

##### Solaris 9/10/11 (OpenCSW)
`pkgutil -i git`
##### Solaris 11 Express
`pkg install developer/versioning/git`

##### OpenBSD
`pkg_add git`

##### Alpine
`apk add git`

### Εγγραφή στο GitHub

Για την εγγραφή σας στο GitHub

1. Μπείτε στην [ιστοσελίδα του GitHub](https://github.com/)
2. Πατήστε το κουμπί **Sign up**
3. Εισάγετε το email σας
4. Εισάγετε έναν κωδικό πρόσβασης
5. Δημιουργήστε ένα όνομα χρήστη
6. Επιλέξτε `y` αν θέλετε να λαμβάνετε ενημερωτικά email από το GitHub και `n` αν δε θέλετε
7. Εισάγετε τον κωδικό μιας χρήσης που είναι στο email σας
8. Πατήστε continue και continue for free
9. Ολοκληρώσατε την εγγραφή σας!
